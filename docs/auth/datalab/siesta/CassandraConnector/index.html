Type Members

  1.  class ApacheCassandraConnector extends DBConnector

    This class handles the communication between between SIESTA and Cassandra.

    This class handles the communication between between SIESTA and Cassandra. It inherits all the signatures of the methods from auth.datalab.siesta.BusinessLogic.DBConnector and overrides the reads and writes to match Cassandra properties. Cassandra is a distributed key-value database. Therefore each record should have a key (either single or complex key).

    See also

    CassandraTables, ApacheCassandraTransformations, where the structure of the tables is described in the first and the transformations between RDDs and Dataframes in the second.

Value Members

  1.  object ApacheCassandraTransformations

    This class is responsible to transform the loaded data from Cassandra into RDDs to processed by the main pipeline.

    This class is responsible to transform the loaded data from Cassandra into RDDs to processed by the main pipeline. Then transform the RDDs to RDDs with specific types, in order to utilize the spark api to store them back in Cassandra. Therefore there are 2 transformations for each table, one to load data and one to write them back.

  2.  object CassandraTables

    This class contains the structure and the names of the tables in Cassandra for a given log database name.

    This class contains the structure and the names of the tables in Cassandra for a given log database name. Additionally it contains the match between the compression algorithms set to the parameters and the compression classes tht needs to be set in Cassandra in order to facilitate compression
